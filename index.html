TÍTULO DE LA PRÁCTICA
"Módulo bluetooth HC-05"

Se realiza una sencilla aplicación con app Inventor para encender 2 focos de 110 voltios controlados con arduino y modulo bluetooth

Introducción

En esta práctica se implementará una conexión entre una placa Arduino UNO y un módulo Bluetooth HC-05, con el objetivo de encender y apagar dos LEDs mediante una aplicación sencilla diseñada en App Inventor. Este proyecto integra conceptos de electrónica básica, programación en Arduino y desarrollo de aplicaciones móviles, permitiendo comprender de manera práctica el funcionamiento de la comunicación inalámbrica vía Bluetooth

Objetivo General

Desarrollar un sistema de control inalámbrico que permita encender y apagar dos LEDs conectados a un Arduino UNO mediante una aplicación móvil creada en App Inventor utilizando el módulo Bluetooth HC-05.

Objetivos Específicos

  • Conocer el funcionamiento básico del módulo Bluetooth HC-05 y su conexión con Arduino.
  • Diseñar una aplicación móvil sencilla en App Inventor para enviar comandos a Arduino.
  • Implementar el código en Arduino que reciba las señales Bluetooth y controle dos LEDs.
  • Comprobar la comunicación inalámbrica entre el dispositivo móvil y el Arduino.
  • Analizar la importancia del control remoto de dispositivos mediante tecnologías inalámbricas.

MARCO TEÓRICO

Conceptos importantes

  • Arduino UNO: Placa de desarrollo basada en el microcontrolador ATmega328P, muy utilizada para proyectos de electrónica y programación.
  • Módulo Bluetooth HC-05: Dispositivo que permite la comunicación inalámbrica entre Arduino y un teléfono u otro dispositivo Bluetooth mediante comunicación serial.
  • Protoboard: Placa plástica con orificios que permite conectar componentes electrónicos sin necesidad de soldar.
  • Programa Proteus: Software de simulación electrónica usado para diseñar y probar circuitos antes de implementarlos físicamente.
  • Focos: Dispositivos eléctricos que convierten energía en luz, usados en proyectos de control de iluminación.
  • Módulo Relé: Componente electrónico que actúa como interruptor controlado electrónicamente, permitiendo manejar cargas de mayor voltaje como focos o motores.
  • Resistencia: Componente que limita el paso de corriente eléctrica para proteger otros dispositivos.
  • Diodo LED: Dispositivo semiconductor que emite luz al paso de corriente en una sola dirección.
  • Sensor: Dispositivo que detecta cambios en el entorno (temperatura, luz, movimiento, etc.) y envía esa información a Arduino.
  • Fuente de Alimentación: Dispositivo que proporciona energía eléctrica estable para alimentar circuitos.
  • App Inventor: Plataforma en línea de Google/MIT para crear aplicaciones móviles de manera visual y sencilla.
  • Comunicación Serial: Método de transmisión de datos bit por bit entre dos dispositivos electrónicos (ej. Arduino y HC-05).
  • Microcontrolador: Circuito integrado que ejecuta programas y controla dispositivos en sistemas electrónicos.
  • Cable Jumper: Cables flexibles con conectores que permiten realizar conexiones rápidas en protoboard.

PROCEDIMIENTO

Código usado en la práctica (IDE ARDUINO )

// Pines usados para los focos (o relés)
int foco1 = 8;
int foco2 = 7;

void setup() {
// Iniciar comunicación con el HC-05
Serial.begin(9600); // Velocidad por defecto del HC-05

// Configurar pines como salida
pinMode(foco1, OUTPUT);
pinMode(foco2, OUTPUT);

// Inicialmente apagados
digitalWrite(foco1, LOW);
digitalWrite(foco2, LOW);
}

void loop() {
// Verificar si llega un dato por Bluetooth
if (Serial.available()) {
char comando = Serial.read(); // Leer un carácter

if (comando == 'A') {
digitalWrite(foco1, HIGH); // Encender foco 1
}
else if (comando == 'B') {
digitalWrite(foco1, LOW); // Apagar foco 1
}
else if (comando == 'C') {
digitalWrite(foco2, HIGH); // Encender foco 2
}
else if (comando == 'D') {
digitalWrite(foco2, LOW); // Apagar foco 2
}
}
}

ANÁLISIS Y DISCUSIÓN

Preguntas y Respuestas de la práctica

1. ¿Qué es el módulo HC-05?
Es un módulo Bluetooth que permite la comunicación inalámbrica entre Arduino y dispositivos móviles.

2. ¿Qué tipo de comunicación utiliza el HC-05?
Utiliza comunicación serial (UART) para enviar y recibir datos.

3. ¿Qué pines se utilizan en Arduino para conectar el HC-05?
Normalmente se conectan los pines TX y RX del módulo a los pines digitales 0 y 1 del Arduino UNO, o se pueden usar pines con SoftwareSerial.

4. ¿Qué voltaje necesita el módulo HC-05?
El pin VCC funciona con 5V, pero sus pines de comunicación (TX/RX) trabajan con 3.3V lógicos.

5. ¿Qué función cumple App Inventor en esta práctica?
Permite crear la aplicación móvil que envía comandos vía Bluetooth al Arduino.

6. ¿Qué comandos se envían desde la app?
Se envían caracteres o cadenas simples como '1' o '2' para encender o apagar cada LED.

7. ¿Cuál es la ventaja de controlar LEDs con Bluetooth?
Permite un control remoto inalámbrico, sin necesidad de cables físicos entre el móvil y Arduino.

8. ¿Qué aplicación práctica tiene este proyecto?
Puede ser la base para desarrollar sistemas de domótica, control de luces y automatización de dispositivos.

9. ¿Qué conocimientos se refuerzan con esta práctica?
Programación en Arduino, manejo de comunicación serial y diseño de apps móviles.

10. ¿Qué mejoras se pueden implementar?
Se podría ampliar el sistema para controlar más dispositivos, agregar sensores o implementar seguridad en la conexión.

Conclusiones de la Práctica

  • El módulo Bluetooth HC-05 permite establecer una comunicación confiable entre un dispositivo móvil y un Arduino.
  • La integración entre App Inventor y Arduino facilita el diseño de aplicaciones prácticas y de bajo costo.
  • El uso de tecnologías inalámbricas es fundamental para el desarrollo de sistemas de automatización y domótica.
  • Esta práctica fortalece el aprendizaje de programación y electrónica aplicada.
